- **ID:** TC-01
- **Приоритет (Priority):** High (P1)
- **Серьезность (Severity):** Critical

- **Описание:** Проверка базовой возможности создания объявления при передаче всех обязательных и валидных полей.
- **Шаги воспроизведения:**
  1. Подготовить JSON-тело запроса с валидными `seller_id` (int), `name` (str), `price` (int) и объектом `statistics`.
  2. Отправить POST-запрос на эндпоинт `/api/1/item`.
  3. Выполнить валидацию схемы полученного ответа.
- **Фактический результат:** Статус `200 OK`, но сервер возвращает строку вместо объекта (BUG-01).
- **Ожидаемый результат:** Статус `200 OK`. Тело ответа — валидный JSON, соответствующий модели ItemResponse.

- **Ожидаемый результат:** Статус `400 Bad Request`. Сервер должен отклонять некорректные типы данных.

## [TC-27] Валидация типов 'sellerId': Отрицательное число (BUG-11)
- **ID:** TC-27
- **Приоритет (Priority):** High (P1)
- **Серьезность (Severity):** Major
- **Компонент:** API / Validation
- **Описание:** Проверка того, что система блокирует или корректно обрабатывает запросы с отрицательными ID продавцов.
- **Шаги воспроизведения:**
  1. Отправить GET запрос на `/api/1/-500/item`.
- **Фактический результат:** Статус `200 OK` и пустой список (BUG-11).

## [TC-28] Валидация типов 'sellerId': Числовое переполнение (Overflow)
- **ID:** TC-28
- **Приоритет (Priority):** Low (P3)
- **Серьезность (Severity):** Minor
- **Компонент:** API / Reliability
- **Описание:** Проверка устойчивости системы при передаче экстремально большого числа в качестве ID продавца.
- **Шаги воспроизведения:**
  1. Отправить GET запрос на `/api/1/999999999999999999999/item`.
- **Фактический результат:** Статус `400` или `404`.
- **Ожидаемый результат:** Статус `400 Bad Request` или `404 Not Found`. Отсутствие ошибки `500 Internal Server Error`.
